The one belief

Every team is building an AI agent. Almost none of them can trust it in production.

The models got good enough to act, not just answer. What is missing is the layer that makes acting safe: permissions, observability and a way to undo. We are building that layer.
The gap

Agents that can act are agents that can break things

How agents ship today
A prompt, a pile of tools and blind faith. No permission model, no record of what the agent did, no way to roll back a bad action. One wrong call hits a customer.
How agents should ship
Scoped permissions per action, a full audit trail of every step, and one-click rollback. The agent moves fast inside a box you control.
How it works

A runtime that wraps every agent action

Aperture sits between your agent and the world. Each action passes through four gates before it touches a real system.
Policy: every tool call is checked against scoped, per-agent permissions before it runs
Simulation: risky actions run in a sandbox first, so the agent previews the outcome
Audit: every step, input and result is recorded as a replayable trace
Rollback: any action, or a whole run, can be reversed in one click
